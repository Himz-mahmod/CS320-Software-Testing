# CS 320 – Software Testing Portfolio

## Overview
This repository contains selected artifacts from CS 320, including Contact Service implementation and unit testing files. These projects demonstrate my understanding of software testing, automation, and quality assurance principles.

## How can I ensure that my code is functional and secure?

I ensure functionality by writing comprehensive JUnit tests that validate all documented requirements. I create both positive and negative test cases to verify expected behavior and proper exception handling. Input validation is enforced in constructors and setter methods to prevent invalid object states. Boundary value testing is used to confirm limits are correctly handled.

## How do I interpret user needs and incorporate them into a program?

I begin by carefully reviewing the requirements specification and translating each constraint into validation rules within the program. Every user requirement is directly mapped to a test case to ensure traceability. By designing services that reflect real-world operations (such as adding, updating, and deleting contacts), I ensure the program aligns with user expectations.

## How do I approach designing software?

I design software by separating responsibilities and maintaining modular structure. Data models are kept independent from service logic. I prioritize encapsulation, validation, and test-driven thinking. Breaking problems into smaller components improves maintainability, readability, and long-term reliability.
